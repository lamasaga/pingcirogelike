这是一个非常经典且关键的游戏系统设计问题！你已经意识到了：“效果定义”（Effect Definition） 和 “效果实例”（Effect Instance / Application Context） 必须分离。

下面我为你提供一个清晰、可扩展、面向未来的架构方案，适用于装备、英雄被动、关卡特性等所有场景。

✅ 核心思想：分离「效果模板」与「效果实例」

1. EFFECT_LIBRARY —— 只定义“模板”
这是你已有的部分，保持不变：
// effectDefs.js
export const EFFECT_LIBRARY = {
  'xp_boost': {
    id: 'xp_boost',
    name: 'Knowledge Seeker',
    desc: 'Increases experience gained from battles.',
    type: 'passive_stat',
    targetStat: 'xpMultiplier',
    levels: [
      { val: 0.05 },
      { val: 0.10 },
      { val: 0.20 },
      { val: 0.35 }
    ]
  },
  // ... 其他效果
};

✅ 这里只描述“这个效果最多有几级，每级数值是多少”，不关心谁用、用几级。

2. 引入「效果实例」（Effect Instance）

每次一个效果被“应用”到某个对象（英雄、装备、关卡）时，都创建一个 EffectInstance，它包含：
- effectId：指向 EFFECT_LIBRARY 的 key
- level：当前使用的等级（0 ~ maxLevel-1）
- （可选）source：来源（如装备ID、技能名、关卡名），用于调试或移除

// 示例：一个 EffectInstance
{
  effectId: 'xp_boost',
  level: 1,        // 表示使用第2级（0-indexed）
  source: 'hero_passive_01'
}

3. 在不同实体中管理自己的 Effect Instances

🧙‍♂️ 英雄（Hero）类
class Hero {
  constructor(config) {
    this.level = 1;
    this.passiveEffects = []; // 存放 EffectInstance
    this._initPassives(config.passives);
  }

  _initPassives(passiveConfigs) {
    for (const pc of passiveConfigs) {
      if (pc.type === 'fixed') {
        // 固定等级，如永远1级
        this.passiveEffects.push({
          effectId: pc.effectId,
          level: pc.fixedLevel, // e.g., 0 (表示Lv1)
          source: hero_{pc.name}
        });
      } else if (pc.type === 'scaling') {
        // 随英雄等级提升（例如：每5级升1级，最多4级）
        this.passiveEffects.push({
          effectId: pc.effectId,
          level: this._calculateScaledLevel(pc), // 初始等级
          source: hero_{pc.name},
          scalingRule: pc.scalingRule // 保存升级规则
        });
      }
    }
  }

  // 当英雄升级时调用
  onLevelUp(newLevel) {
    this.level = newLevel;
    // 更新所有可升级的被动
    for (const inst of this.passiveEffects) {
      if (inst.scalingRule) {
        inst.level = this._calculateScaledLevelFromRule(inst.scalingRule);
      }
    }
    this.recalculateStats(); // 重新计算属性
  }

  _calculateScaledLevelFromRule(rule) {
    // rule 示例: { maxLevel: 4, step: 5 } → 每5级升1级，最多4级
    return Math.min(
      Math.floor((this.level - 1) / rule.step),
      rule.maxLevel - 1
    );
  }
}

英雄配置示例（JSON 或 JS 对象）：
const heroConfig = {
  passives: [
    {
      name: '永恒智慧',
      effectId: 'xp_boost',
      type: 'fixed',
      fixedLevel: 0 // 永远是 Lv1 (+5%)
    },
    {
      name: '战斗大师',
      effectId: 'score_master',
      type: 'scaling',
      scalingRule: { maxLevel: 4, step: 6 } // 每6级升1级，最多4级
    }
  ]
};

🛡️ 装备（Item）类
装备通常等级固定（由装备品质决定），所以很简单：

class Item {
  constructor(itemId) {
    this.id = itemId;
    this.effectInstance = null;

    const itemDef = ITEMS[itemId];
    if (itemDef.effectId) {
      // 假设装备的等级由其 rarity 决定，或直接写死
      const effectDef = EFFECT_LIBRARY[itemDef.effectId];
      let level = 0; // 默认Lv1
      if (itemDef.rarity === 'epic') level = 2; // 史诗装备给Lv3

      this.effectInstance = {
        effectId: itemDef.effectId,
        level: level,
        source: item_{itemId}
      };
    }
  }
}

🌍 关卡（Level / Dungeon）特性
关卡也可以有自己的效果列表，比如“毒雾区域”：

class Level {
  constructor(levelId) {
    this.effects = [
      {
        effectId: 'poison_debuff', // 假设有负面效果
        level: 1, // 中等毒性
        source: level_{levelId}_hazard
      }
    ];
  }
}

4. 统一的属性计算器（Stat Calculator）

现在，无论效果来自英雄、装备还是关卡，你都可以统一处理：

function calculateTotalStats(hero, equippedItems, currentLevel) {
  const allInstances = [
    ...hero.passiveEffects,
    ...equippedItems.map(item => item.effectInstance).filter(Boolean),
    ...(currentLevel?.effects || [])
  ];

  const stats = {
    xpMultiplier: 1.0,
    crystalMultiplier: 1.0,
    // ... 其他基础值
  };

  for (const inst of allInstances) {
    const def = EFFECT_LIBRARY[inst.effectId];
    if (!def || inst.level >= def.levels.length) continue;

    const value = def.levels[inst.level].val;
    const statKey = def.targetStat;

    if (def.type === 'passive_stat') {
      // 假设都是加法叠加（你也可以支持乘法、替换等）
      stats[statKey] = (stats[statKey] || 0) + value;
    }
  }

  return stats;
}

✅ 优势总结
需求   如何满足
英雄被动固定等级   使用 type: 'fixed' + fixedLevel

英雄被动随等级成长   使用 type: 'scaling' + 自定义升级规则

装备效果固定   装备构造时根据品质/类型决定 level

关卡负面效果   关卡对象持有自己的 EffectInstance 列表

未来扩展   只需在 EFFECT_LIBRARY 加新效果，无需改核心逻辑

🔮 进阶建议（可选）

1. 支持动态效果：比如“击杀后临时+1级”，可以用 temporaryEffects 数组 + 过期时间。
2. 效果堆叠规则：相同效果是否叠加？用 stackingGroup 字段控制。
3. UI 显示：每个 EffectInstance 可附带 displayName，方便在 tooltip 中显示“Lv2 战斗大师”。
