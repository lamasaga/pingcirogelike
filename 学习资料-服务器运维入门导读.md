## 1. 一次访问从浏览器到页面： mental model

　　简化链路：

```text
浏览器 → 公网 IP + 端口（80/443）→ 云厂商安全组/防火墙放行
      → 本机监听进程（通常是 Nginx）→ 按规则分发：
           · 静态文件（拼词肉鸽 /pingchi）
           · 或转发到本机某端口上的程序（如 Node 3780）
```

　　要点：

- **端口**：`80` 一般是 HTTP，`443` 是 HTTPS；「谁在监听」用 `sudo ss -tlnp | grep ':80\|:443'`（你已用过）。
- **Nginx**：多半是对外的「前台」，负责 TLS、路径、静态目录、`proxy_pass` 到后端。
- **应用进程**：TechVenture 这类要有进程常驻；纯静态拼词项目只要磁盘上有文件即可。

## 2. Linux / Ubuntu：你需要的最小集合

| 概念 | 含义 | 和你工作的关系 |
|------|------|----------------|
| **用户与权限** | `ubuntu` 是普通用户；装软件、改 Nginx 配置常要 `sudo` | 克隆到 `/srv` 可能要 `chown`/`chmod` 让 Nginx 能读 |
| **路径** | `/etc` 系统配置，`/var/log` 日志，`/srv` 常见放业务代码 | 拼词在 `/srv/pingchi`，TechVenture 在 `/srv/techventure` |
| **包与管理** | `apt` 装软件；本项目静态站不强制 Node | TechVenture 服务器上要 `npm i` / `npm run build` |

　　Ubuntu 官方入门可参考：[Ubuntu Server 文档](https://ubuntu.com/server/docs)。

## 3. SSH：远程登录与密钥

　　**SSH** 让你在本地终端「远程操作服务器 shell」。认证常见两种方式：**密码**（不推荐长期使用）、**公钥**（推荐）。

- 私钥在本机（如 `~/.ssh/*.pem` 或 `*_ed25519`），公钥在服务器用户目录 `~/.ssh/authorized_keys`。
- `Permission denied (publickey)` 通常表示：密钥不对、用户不对、或服务器未登记你的公钥。

　　OpenSSH 说明可参考：[OpenSSH 手册（客户端）](https://man.openbsd.org/ssh)。

## 4. Nginx：你最该理解的四件事

### 4.1 配置从哪里来

　　Ubuntu 常见布局：

- 主配置：`/etc/nginx/nginx.conf`（末尾常会 `include sites-enabled/*`）
- 站点片段：`/etc/nginx/sites-available/`（真实文件），`sites-enabled/` 里是 **符号链接**，启用哪个站点就链哪个。

### 4.2 `server { }`：一台站点的一种访问方式

　　每个 `server` 块通常对应：**监听端口 + `server_name`（域名或 `_`）**，里面再写多个 **`location`**。

### 4.3 `location`：路径怎么映射

- **`root`**：URL 路径 **追加** 到目录后面。
- **`alias`**：把某 URL 前缀 **替换** 为磁盘路径（适合 `/pingchi/` 映射到 `/srv/pingchi/`）。
- **`proxy_pass`**：转发给后端 `http://127.0.0.1:端口`（TechVenture 典型）。

### 4.4 改完一定要做的事

```bash
sudo nginx -t          # 语法检查，必须通过
sudo systemctl reload nginx   # 热重载配置（比 restart 温和）
```

　　官方入门：[nginx beginners guide](https://nginx.org/en/docs/beginners_guide.html)（英文，结构清晰）。

## 5. systemd：为什么 TechVenture 要 `restart`

　　**systemd** 管理「服务」的开机自启、崩溃重启、环境变量与日志收集。

- `systemctl status 服务名`：是否在跑、最近日志摘要。
- `journalctl -u 服务名 -e`：滚日志排错。
- 改代码后通常：**拉代码 → 构建 → restart 服务**；纯静态站 **不需要** 这一步。

　　入门：[systemd 概述（ freedesktop ）](https://www.freedesktop.org/software/systemd/man/systemd.html)（偏手册向，可查单元类型）。

## 6. Git 在服务器上的角色

　　服务器上的 `/srv/...` 多是 **`git clone` 出来的工作副本**。日常更新：

```bash
cd /srv/某项目
git pull origin main
```

　　注意：**服务器上若有仅存在于本地的数据**（比赛 JSON、`.env`）不应依赖「忘了备份就重装」；持久化路径要与仓库分开心里有数。

## 7. 排错顺序（建议固化成习惯）

1. **浏览器现象**：空白页 / 404 / 502 / 证书错误。
2. **本机 curl**：`curl -I http://127.0.0.1/路径` 区分是 Nginx 问题还是外网/防火墙。
3. **Nginx**：`sudo nginx -t`、`sudo tail -n 100 /var/log/nginx/error.log`。
4. **后端进程**：`systemctl status`、`journalctl -u`、`ss -tlnp | grep 端口`。
5. **权限与路径**：403 常和目录权限、`alias`/`root` 写错有关。

## 8. 安全与运维意识（小范围服务也要知道）

- **HTTP**：明文传输；仅限可信网络或演示；对外长期服务优先 **HTTPS + 域名**。
- **防火墙 / 安全组**：只放行需要的端口与来源 IP。
- **备份**：比赛数据、数据库、上传文件——按「丢了能不能接受」决定备份频率。
- **不要把私钥、密码提交到 Git**。
