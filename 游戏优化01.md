# 拼词肉鸽 (Word Roguelike) - 游戏优化设计文档 v1.0

## 1. 视觉升级：动态手绘风格背景 (PC端)

为了在不同分辨率下保持“手绘风格”与“UI操作区”的完美融合，不能简单地使用一张大图作为 `background-image`，否则窗口拉伸时元素会对不齐。

### 1.1 设计理念：旅店桌面 (The Tavern Tabletop)
将整个屏幕模拟成一个奇幻世界旅店的木质桌面。
*   **基础层**：木纹纹理，营造沉浸感。
*   **UI 容器层**：现在的 Grid（网格）、Shop（商店）不再是悬浮的白色方块，而是设计成“羊皮纸”、“旧账本”或“魔法石板”的样式。
*   **装饰层**：在 UI 的空隙处（通过 CSS 定位）随机散落装饰物（如：羽毛笔、墨水瓶、金币堆、咖啡渍），增加生活气息。

### 1.2 技术实现方案：分层布局 (Layered Layout)
不要试图用背景图去“框”住 UI，而是让 UI 自带背景。

1.  **全局背景**：使用可平铺 (Repeatable) 的高精度手绘木纹。
2.  **区域背景 (Container Background)**：
    *   `Grid Section` -> 变为一张 **"魔法地图/棋盘垫"** 的图片。
    *   `Sidebar (Shop/Stats)` -> 变为一本打开的 **"冒险者账本"**。
3.  **响应式适配**：
    *   使用 CSS `aspect-ratio` 锁定背景图比例。
    *   使用 `Grid` 布局将装饰性元素（如桌角的蜡烛）固定在特定的 Grid Area，而不是绝对定位，防止遮挡交互区。

## 3. 数值与难度机制完善

目前高难度无法通关的原因是：**目标分数呈指数增长，但玩家的得分能力呈线性增长**。

### 3.1 目标分数公式调整
*   **旧公式**：`Target = Base * (1.5 ^ Round)` (增长太快)
*   **新公式 (建议)**：
    *   `简单`: `30、70、120、180、250、350、500、700、1000、1400、2000、2500、3000、3500、4000` 
    *   `普通`: `40、100、200、350、500、750、1250、1600、2000、2500、3100、3800、4600、5200、6000` 
    *   `地狱之路`: `150、300、500、800、1300、2000、2500、3000、3600、4200、5000、6000、7000、8000、9000` 

### 3.2 经济平衡
*   **增多刷新次数**：每回合首个商店刷新免费。
*   **通关奖励**：
    *  基础 7 块钱，如果你本回合没有购买任何字母，额外增加1块钱。

## 4. 新模式：

### 1 地狱之路
 相对于普通难度，地狱之路开局的金额有21块钱，且有3次免费刷新机会，遗物的获取从三选一变为三次刷新获取的机会（每次只有一个遗物让玩家选择是获取还是刷新掉，刷新后出现一个不重复的新遗物，然后玩家继续判断是否获取，每一轮共有3次刷新机会）


### 2 巨人之路 (Path of Giants)

这是一个区别于普通闯关的**快速随机模式**。

### 4.1 核心规则
1.  **取消关卡**：没有 15 关限制，直接进入最终阶段，并且没有目标分数，完成游戏后的得分进入排名
2.  **一次性获取遗物**：开局的时候直接连续4次获得遗物，然后直接开始仅一次的游戏；
3.  **随机获得30张牌**：取消商店区，界面合并商店区和备选区，成为一个完整的字母储备区，再游戏开始时直接随机获得30个字母（确保元音字母和常用辅音字母至少有一个，其它字母随机获取，但一个字母获得不会好过3次）

### 4.2 积分榜
*   记录玩家的“最高关卡数”和“总得分”。
*   由于没有后端，暂存储在 `localStorage` 中展示“历史最高记录”。

## 5. 操作 UI 与体验优化 (UX Improvements)

### 5.1 拼词盘快照 (Snapshot)
*   **痛点**：玩家经常想试一种摆法，发现分不高，想撤回却忘了原来怎么摆的。
*   **方案**：在 Grid 下方增加三个小按钮 `[保存状态]` `[恢复状态]`。
    *   技术上使用 `JSON.stringify(gridSystem.grid)` 深拷贝暂存数据。

### 5.2 备选区 (Storage) 扩容与修复
*   **Bug修复**：目前的 Storage 可能是基于数组索引硬编码的，需要改为动态列表 (`flex-wrap`)。
*   **扩容**：将上限提升至 12 或无限（带滚动条）。
*   **交互**：支持从 Grid 直接右键点击字母将其送回 Storage（快速收纳）。

### 5.3 智能悬停提示 (Smart Tooltip)
*   **痛点**：玩家不知道为什么这个词得了 100 分。
*   **方案**：
    *   当鼠标悬停在网格上的单词或字母时，弹出一个浮窗。
    *   **显示详情**：
        ```text
        单词: APPLE
        基础分: 8
        ----------------
        长度加成 (5): x2.0
        遗物 [元音护符]: +4
        交叉加成: x2.0
        ----------------
        总计: 48 分
        ```
    *   这需要重写 `GridSystem.calculateScore` 方法，使其能返回计算过程的日志，而不仅仅是最终数字。
